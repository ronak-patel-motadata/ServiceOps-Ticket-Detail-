Design a professional, modern, first-time in-app guide overlay for the Ticket Details page using the attached screenshots as the visual reference and UI source of truth.

Context:
This is a guided product tour that appears only when a user opens the ticket details page for the first time. The guide should help users quickly understand the layout and how to work inside the page. The experience must feel polished, lightweight, modern, and enterprise-grade.

- Keep the existing Ticket Details page UI intact.
- Add the onboarding guide on top of the current screen as a contextual step-by-step walkthrough.
- Follow the popup design direction from “In app guide Popup.png”, but make it more refined, cleaner, and more premium.
- The result should be very user friendly, easy to scan, and aligned with current SaaS product design trends.

What to create:
Create a multi-step in-app guided tour for the Ticket Details page with:
1. Highlighted focus area for each step
2. Dimmed background overlay
3. Clean floating tooltip / popup card
4. Step counter like 1/4, 2/4, etc.
5. Primary CTA: Next / Continue
6. Secondary CTA: Skip tour
7. Optional Back button from step 2 onward
8. Smooth visual flow between steps
9. Clear anchor positioning so the tooltip points to the correct UI area
10. Minimal obstruction of the user’s working area

Design style:
- Modern B2B SaaS
- Clean, minimal, premium
- Soft shadows
- Rounded corners
- Balanced whitespace
- Crisp typography
- Subtle highlight treatment around the target area
- Accessible contrast
- Lightweight onboarding feel, not bulky or distracting
- Professional and calm visual tone

Tour flow:
Step 1: Main work area
Target area:
Highlight the left content section of the ticket page where the ticket description, conversation, tasks, approvals, relations, audit trails, and resolution tabs are visible. Also show that the reply input stays fixed at the bottom.

Popup title:
Your main workspace

Popup text:
Work from the left panel to review the ticket, switch between work tabs, and stay on top of conversations. The reply box stays fixed at the bottom, so actions are always within reach.

Step 2: Right icon sidebar
Target area:
Highlight the vertical icon sidebar on the right edge.

Popup title:
Quick access sidebar

Popup text:
Use this sidebar to switch between supporting views like Ticket Properties, Activity & Resources, and AI Suggestions without leaving the ticket.

Step 3: Ticket properties panel
Target area:
Highlight the Ticket Properties area and its controls like search, filter, and pin.

Popup title:
Ticket details, your way

Popup text:
View ticket fields, additional fields, system fields, and custom fields in one place. Search, filter, or pin important fields to keep the details you need on top.

Step 4: AI assistant
Target area:
Highlight the AI input / AI section at the bottom right and relevant AI assistance area.

Popup title:
AI help, built in

Popup text:
Use AI to summarize the ticket, suggest next steps, draft responses, and help update the ticket faster.

Content rules:
- Keep each popup short and easy to read
- Do not use long paragraphs
- Make the language simple, confident, and product-friendly
- Avoid jargon overload
- Focus on clarity and onboarding value

Interaction behavior:
- Show one step at a time
- Each step should clearly highlight the relevant section
- Clicking Next moves to the next focused area
- Clicking Skip closes the tour
- The tour should feel guided, not intrusive
- Maintain good tooltip placement so it never blocks the exact thing being explained

Extra expectations:
- Make the onboarding visually consistent with the existing ticket page
- Add subtle motion/micro-interaction suggestions if possible
- Ensure the popup looks better and more modern than the provided “In app guide Popup.png”
- Optimize for enterprise users who need to understand the page fast
- The final result should look ready for production, not like a rough concept
